# 05. 정책 적용

이 단계는 4단계에서 만든 정책 미반영 적정가격에 국내 정책 효과를 반영하는 단계입니다. 원본 Colab의 `3단계-v2: 국내 정책 적용` 구간을 분리했습니다.

핵심 목적은 유류세 인하 정책을 주유소 적정가격과 적정 가격대에 수치 반영하고, 최고가격제는 주유소 가격에서 직접 차감하지 않고 정유사 weekly 결과에서 별도 점검하는 것입니다.

## 실행 파일

- `05_policy_application.ipynb`: Colab에서 단독 실행 가능한 정책 적용 노트북

노트북의 기본 경로는 원본 작업 경로를 그대로 유지했습니다. 다른 사용자는 아래 경로만 본인 Drive 구조에 맞게 수정하면 됩니다.

```python
ROOT_PATH = "/content/drive/MyDrive/Data_analysis/The appropriateness of domestic oil prices compared to international oil prices/산업부/"
DATA_PATH = ROOT_PATH + "data/"
PROCESSED_PATH = ROOT_PATH + "preprocessed_data/"
OUTPUT_BASE = Path(ROOT_PATH) / "정책적용_v2"
```

4단계 production CSV에는 `date`, `pred_gross`, `band_low`, `band_high`가 반드시 있어야 합니다. `prediction_구조검증_v2.csv` 기준 필수 컬럼과 band 순서 검증은 휘발유와 경유 모두 `OK`입니다.

## 정책 적용 방식

노트북의 정책 적용 옵션은 아래와 같습니다.

```python
BASELINE_INPUT_MODE = "prediction_is_already_nonpolicy"
FAIR_SHIFT_MODE = "official_tax_cut_vat_inclusive"
RETAIL_PASS_THROUGH_RATE = 1.0
RETAIL_PASS_THROUGH_LAG_DAYS = 0
NUMERICALLY_APPLY_POLICY_CATEGORIES = [0]
BASE_FUEL_TAX_EXVAT_원L = {
    "휘발유": 745.89,
    "경유": 528.75,
}
VAT_RATE = 0.10
```

해석하면 다음과 같습니다.

- 4단계의 `pred_gross`, `band_low`, `band_high`는 이미 정책 미반영 기준으로 간주합니다.
- 카테고리 0인 유류세 인하율만 주유소 적정가격과 band에서 수치 차감합니다.
- 유류세 인하 효과는 VAT 포함 금액 기준입니다.
- 전가율은 1.0, 반영 지연은 0일입니다.
- 카테고리 1인 최고가격제는 주유소 적정가격에서 직접 차감하지 않고 정유사 단계 감사표로만 점검합니다.

정책 적용 후 컬럼은 아래 관계를 가집니다.

```text
적정가격_미정책_원L = pred_gross
적정범위_미정책_하한_원L = band_low
적정범위_미정책_상한_원L = band_high

정책효과_원L = 정책적용_시프트_원L
적정가격_정책적용_원L = 적정가격_미정책_원L - 정책효과_원L
적정범위_정책적용_하한_원L = 적정범위_미정책_하한_원L - 정책효과_원L
적정범위_정책적용_상한_원L = 적정범위_미정책_상한_원L - 정책효과_원L
```

## 확인된 주의점

1. 적정가격과 band에는 실제 0값이 없습니다. 다만 4단계 rolling 예측 불가 구간은 5단계에서도 빈 값으로 이어집니다. 웹 표시나 다음 모델 입력에서는 `적정가격_정책적용_원L`, `적정범위_정책적용_하한_원L`, `적정범위_정책적용_상한_원L`의 결측 여부를 먼저 확인해야 합니다.
2. 2008년 유류세 10% 인하 기간 261일은 정책 기간으로 표시되지만 4단계 예측값이 없어 평균 정책효과가 비어 있습니다. 정책 적용 가능 여부와 실제 효과 계산 가능 여부를 분리해서 봐야 합니다.
3. 최고가격제 2차 정책과 2026-03-27 이후 유류세 인하 확대 정책은 정책 이력에는 있으나 현재 데이터 종료일이 2026-03-18이라 일별 결과에는 반영되지 않았습니다.
4. 최고가격제는 주유소 적정가격에 직접 반영하지 않습니다. 웹에서 이를 표시할 때는 `가격 차감 정책`이 아니라 `정유사 상한 점검 정책`으로 분리해야 합니다.

## 검토 결론

5단계 결과 파일은 코드 산출물 구조와 일치하며, 이 레포의 `outputs/`에는 필요한 결과 CSV와 PNG를 모두 반영했습니다. 결과 내용도 4단계 `pred_gross` 연결, 유류세 정책 차감, 범위 판정 변화, 최고가격제 별도 감사 측면에서 정합성이 확인됩니다.

다만 초기 예측 불가 구간의 결측, 2008년 정책효과 결측, 2026-03-27 이후 정책의 데이터 구간 밖 상태, 최고가격제의 비수치 반영 원칙은 후속 웹 자동화와 모델 입력 단계에서 반드시 명시적으로 처리해야 합니다.
