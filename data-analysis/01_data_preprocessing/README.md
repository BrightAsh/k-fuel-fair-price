# 01. Data Preprocessing

01 단계는 분산된 원천 데이터를 일별 분석 테이블로 통합하는 단계입니다. 이후 02~05 단계의 시차 분석, benchmark 산출, 적정가격 계산, 정책 적용 판단은 모두 이 전처리 산출물을 기준으로 수행됩니다.

## 단계 목적

| 목적 | 내용 |
|---|---|
| 날짜 기준 통합 | 원유, 제품가격, 환율, 전국 주유소 가격, 세금을 같은 날짜 축으로 결합 |
| 유종별 분석 구조 생성 | 휘발유와 경유를 같은 스키마로 정리 |
| 국내 반영 가격 산출 | 국제 가격과 환율을 국내 원화 기준 후보 가격으로 변환 |
| 정책/시차 분석 준비 | 세금, 소비자 가격, 정유사 공급 가격을 함께 비교할 수 있게 정리 |

## 입력 데이터

| 입력 | 분석상 의미 |
|---|---|
| 원유 가격 | 국제 원유 비용 흐름 |
| 국제 석유제품 가격 | 휘발유/경유의 직접적인 국제 기준 가격 |
| 원/달러 환율 | 국제 가격을 국내 원화 기준으로 변환 |
| 전국 평균 주유소 가격 | 소비자가 실제로 보는 국내 판매 가격 |
| 브랜드별 평균 가격 | 유통/브랜드별 가격 차이 후보 |
| 유류세 추이 | 세금 구성과 정책 기간 반영 |
| 정유사 주간 공급 가격 | 국제 가격이 국내 공급 가격으로 반영되는 중간 단계 |

## 처리 로직

### 1. 날짜 정규화

각 원천 파일의 날짜 컬럼명이 다르므로, 날짜 후보 컬럼을 탐색해 `date`로 통일합니다. 날짜 해석이 불안정한 행은 제외해 이후 단계의 시차 계산이 흔들리지 않도록 합니다.

### 2. 국제 가격 원화 환산

국제 제품가격은 달러 기준이므로 환율을 결합해 원화 기준 후보 가격으로 변환합니다. 이 값은 실제 국내 가격과 직접 비교하기 위한 기초 feature입니다.

### 3. 유종별 컬럼 정리

휘발유와 경유를 같은 구조로 다루기 위해 유종별 실제 가격, 국제 기준 가격, 세금, 정유사 공급 가격 컬럼을 표준화합니다.

### 4. 정유사 공급 가격 주간 데이터 처리

정유사 공급 가격은 주간 단위이므로, 일별 분석 테이블과 결합할 수 있도록 날짜 기준으로 확장하거나 가장 가까운 기준일 값을 사용합니다.

### 5. 결측/중복 정리

동일 날짜에 여러 값이 들어온 경우 평균 또는 최신 값을 사용하고, 핵심 분석에 필요한 가격 컬럼이 없는 행은 이후 단계에서 제외될 수 있도록 결측 상태를 유지합니다.

### 6. 통합 테이블 저장

최종 통합 데이터를 02~05 단계가 공통으로 쓰는 일별 분석 테이블로 저장합니다.

## 주요 산출물

| 산출물 | 내용 |
|---|---|
| `분석용일별통합데이터.csv` | 날짜별 국제 가격, 환율, 전국 평균 가격, 세금, 정유사 공급 가격을 결합한 분석 기준 테이블 |

## 분석 의미

01 단계는 가격 예측이나 적정가격을 직접 산출하지 않습니다. 대신 다음 질문에 답할 수 있는 형태로 데이터를 맞춥니다.

| 질문 | 다음 단계 |
|---|---|
| 국제 가격 변화가 국내 가격에 며칠 뒤 반영되는가? | 02 |
| 국제 가격과 국내 가격을 비교할 때 어떤 benchmark가 적절한가? | 03 |
| 세금과 국제 가격을 고려한 적정가격은 얼마인가? | 04 |
| 정책 기간에는 어떤 보정이 필요한가? | 05 |

## 다음 단계 연결

02 단계는 이 통합 테이블을 이용해 국제 가격과 국내 가격 사이의 최적 시차를 찾습니다. 03~05 단계는 02의 시차 결과를 반영해 benchmark와 적정가격을 계산합니다.
